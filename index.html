* 如果參與這個計畫，會怎麼安排學習時間？
學習時間規劃

以在職身份參與勢必會比較辛苦，但我會努力堅持。比起完全沒技術背景的人，我相信我有多那麼一點點的優勢，因此在前期或許可以超前進度，為後面難度更高的課程爭取多一點時間，也會更有效率的投入時間在不熟悉或全新的內容。因為平日白天要工作，因此學習會以平日晚上以及週末為主，若要明確的時數分配，則是週末兩天各 12 個小時，平日則每天抽出 3 個小時。去年有報名參加其他平台提供的資料分析課程，因此滿習慣這種整個週末都要學習、寫作業的生活。

* 請描述一件你曾經碰到最困難的事情，如何克服？

我認為最困難的事情就是堅持。我從去年開始為自己訂下學習的規劃，然而這些學習不見得可以直接運用在工作上，更多是打基礎課程，因此相對於工作，急迫性不高，也就是說不會有任何人逼我一定要完成這些學習目標，未達標也不會有任何懲罰。再加上現在誘惑太多，有太多娛樂活動可以選擇，閒暇之餘要能抵擋得了這些誘惑而能持續不斷的學習，我認為是需要高度自律和定力的。而我從去年開始，我定下每年至少完成五堂課程，雖然去年到現在沒有過很久，但目前仍持續在朝這個目標前進。已經上過的課程包含：英文口說課、資料結構、統計推論、推薦系統、簡報課、商業模式。雖然這些課是我認為有幫助或多少感興趣的課程，但終究會有枯燥、困難的段落，因此能夠撐過最不有趣的部分，是我覺得很有成就感的一件事。我自己克服惰性、抵抗人性的方式，就是軟硬兼施。達標時要給自己獎勵，未達標則要懲罰自己，同時要營造出一個舒適但卻無法偷懶的環境，也是能讓自己持續堅持很重要的條件。

* 關於這份申請網頁，請分享一個你開發時的技術心得。

在設計階段，總會對結果有很美好的想像，比如要有什麼花俏的功能、排版要多好看等，但在開發階段若要一次到位，經驗、能力又還不純熟的情況下，常常造成卡關，最終導致花了很多時間，卻一點成果也沒有，也造成心情很沮喪。因此，心態調整好，把專案的工作切成多個小小的任務，逐個完成，且過程中先追求有再追求好，功能層層堆疊、優化，才能開發出好的產物。

在這份申請頁中，我想加入一些互動的環節，比如按了按鈕後，顯示文字，但 html 無法處理動態內容，因此我額外加入了 javascript code。不過，更好的寫法是把 javascript code 獨立成其他檔案。
